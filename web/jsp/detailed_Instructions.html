Rescue is about saving a prisoner on the given maze. The idea is that the participant will have to program the ‘Player’ logic. There are four levels in the game, which are listed below or in the level details tab.
The player will move along the blocks to rescue the captive. While in use a player's position will be represented as P and a captive will be represented as G. Through the game a player can move anywhere in/across the white block and must avoid the walls. In the further levels, the player have to collect few keys represented by K before reaching the captive. Further the player have to collect the keys, avoid the enemies (represented by E) and save the captive
Difficulty level increases with the level:
  1. Save the prisoner with minimum number of moves.
  2. Save the prisoner after collecting keys.
  3. Save the prisoner after collecting keys & avoiding one enemy marching on the way.
  4. Save the prisoner after collecting keys & avoiding two enemies guarding the keys.
The following are a few important tips you should be aware of before you begin the coding contest:
For Java
•   A player is allowed to write his code only on the UserGame.java file (present in java_code.zip)
•   Any other file except UserGame.java are support files and should not be changed
•   GameInterface.java is provided to assist the player in writing the code
•   Player can call any function from the class GameInterface and use the result in his code. (click here)
•   makeMove(command,type) is the function used to move the player position (up/down/right/left).
•   A player’s position will get updated only when this function is called with proper parameters
    example: ob.makeMove("move","down")
For C
•   A player is allowed to write his code only on the user_game.c file (present in c_code.zip)
•   Any other file except user_game.c are support files and should not be changed
•   game_interface.c is provided to assist the player in writing the code
•   Player can call any function from game_interface.c and use the result in his code. (click here)
•   make_move(command,type) is the function used to move the player position (up/down/right/left).
•   A player’s position will get updated only when this function is called with proper parameters
    example: make_move("move","down")
We request you to follow the steps listed below in order to begin the game:
•   A user will have to submit a valid email id to begin the game
•   Once logged in the user would need to activate the game by clicking on the button  
•   When clicked the game board will open - Keep the board open
•   Download the appropriate code.zip file and extract it in any directory in your computer
•   Compile the extracted file from the above folder by following command
     For Java,   javac com/devsquare/fts/*.java
     For C,   gcc -o user_game user_game.c
•   Run your code by following command
     For Java,   java -cp "." com.devsquare.fts.UserGame "sessionkey"
     For C,   ./user_game "sessionkey"
•   You will find a unique sessionkey on the same page
•   If you successfully reach the prisoner the board will be refreshed to the next level

Go through the following tips before starting the game:
•   Backtracking and typical maze traversals algorithms are not very helpful in this game. You should try tweaking with these     algorithms.
•   In level 2, first your goal should be to collect the keys than traveling the shortest path. But you will earn more points if you     travel the shortest path possible and collect all the keys.
•   In level 3 and 4, you should try to avoid the enemies than go for collecting keys.
•   Solving all the levels successfully is recommended than solving few levels optimally.
•   In level 3 & 4, try to be as far from the enemies as possible.
•   You cannot move to the next level without solving previous level.
•   Try writing a common program for all the stages, as it will save your time.
•   Try to buildup on your code for level 1 for the solution of next levels.
•   You can attempt as many times as you like, so don’t be afraid of being wrong few times.
•   Utilize the provided functions intelligently to reduce your programming effort.

The following are a few important tips you should be aware of before you begin the coding contest:
Server Issues:
  • Game Board not loading properly or not loading at all.
    • Check the version of Java Runtime Environment you have. It should be jre1.6 or above.
    • Clear the java cache. Go to Control Panel -> Java -> General -> Temporary Internet Files -> Delete Files. Uncheck ‘Keep temporary files on my computer’.
    • Check java network settings. Go to Control Panel -> Java -> Network Settings. Get your network settings checked by your network administrator.
    • If the problem still exists please contact the administrator.
Client Software Issues:
  • Host website is not letting you register. Here are the few genuine cases when website will not allow you to register:
    • If your first name or last name contains numbers or special characters.
    • If your email id is not valid.
    • If you are already registered by the same email id.

      • (If any one of the above cases does not apply, please contact the administrator)
  • Forgot password is not sending you a mail containing your password:
    • Check if you email id is valid and active.
    • You will not receive any mail of you are not registered.
    • Try again after sometime, if the problem still exists please contact administrator
  • Not able to run the code.
    • You should have jdk1.6 or above for java test and gcc compiler for C test in your PC.
    • Keep the support code in the directory structure as instructed.
    • Users taking C test are expected to use linux and latest gcc compiler.
    • Check with the network administrator if the port 443 is open for TCP/IP traffic.
    • If the problem still persists, please contact administrator.
Connection Issues:
  • Host website not responding properly.
    • Try to ping the website. Open a command prompt. Type ping "website name", it should respond with replies from the corresponding IP address. If not the server is down, please wait for sometime or contact administrator.
    • Check if internet is working by opening some other website.
    • Enable the javascript of your browser (if disabled) for better results.
    • If the problem still exists, please contact administrator.
  • If the game is not responding to your code.
    • If you think you have changed any of the support files from java_code.zip or c_code.zip please download it again from the website and replace it with the old files.
    • If problem still exists, please contact administrator.
Support Issues:
  • If you find any bug in the website or in game, go to ‘contact’ tab, enter your details, your problem, and select ‘report bugs’ from drop down menu and post. You request will be forwarded to the developers and we will get back to you soon with a solution.
  • If you have any general query, feel free to follow the same procedure as above but from drop down menu select ‘general queries’. Your mail will be forwarded to the relevant personnel.
